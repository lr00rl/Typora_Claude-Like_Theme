# Release Notes - v1.3.0

## Code Block Line Numbers: Selection & Rendering Optimization

This update completely overhauls the code block line number system in Typora to resolve long-standing issues with selection highlights and lazy-rendering delays.

### 1. Selection & Cursor Isolation (The "Blue Overlay" & "Wall" Fix)
- **Problem**: In previous versions, selecting code would cause a blue highlight to cover the line number area, and the cursor could be moved into the gutter.
- **Solution**: 
    - Restored `margin-left: 2.75em` on the `.CodeMirror-sizer`. Because CodeMirror's internal logic respects this margin, the cursor now "hits a wall" at the vertical divider and cannot enter the line number area.
    - Positioned custom line numbers using `left: -2.75em` within that margin.
    - Added `clip-path: inset(0 0 0 0)` on the `.CodeMirror-selected` element as a safety layer (though the margin now handles most selection isolation naturally).
- **User Select**: Maintained `user-select: none` on the line number wrapper and `user-select: text` on the code lines to ensure line numbers are never copied to the clipboard.

### 2. Rendering Strategy: Native Gutter vs. CSS Counters
- **Problem**: Typora's native CodeMirror gutter is lazy-rendered, causing line numbers to appear with a 1-2 second delay during fast scrolling.
- **Solution**: 
    - Disabled the native gutter (`.CodeMirror-gutters`) entirely.
    - Switched to a **CSS Counter** based approach (`counter-increment: css-line-number`).
    - Line numbers are now generated instantly by the browser's CSS engine, eliminating all scrolling lag.

### 3. Visual & Structural Improvements
- **Vertical Divider**: Added a subtle `1px` vertical line (`border-right`) at the 2.75em mark. This provides a clear visual anchor for the start of the editable code area, separating the line numbers from the content.
- **Z-Index Layering**: Added `z-index: 10` to line numbers to ensure they stay on top of any background layers while maintaining the `clip-path` isolation.
- **Precise Alignment**: 
    - Standardized `font-size: 1em` and `line-height: 1.55` to match the code text perfectly.
    - Used `width: 2.75em` and `padding-right: 0.75em` with `box-sizing: border-box` for pixel-perfect spacing.
    - Used `opacity: 0.38` for a subtle, non-distracting appearance that adapts to both light and dark themes.

### 4. Theme Consistency
- Applied identical logic to both `claude-like.css` (Light) and `claude-like-dark.css` (Dark) to ensure a unified experience across all environments.
